# デザインガイドライン

このドキュメントは、`src/app/globals.css` およびコンポーネントでの余白・レイアウトの使い方を定めます。統一感と汎用性を保つため、新しいスタイルを足すときはここに従ってください。

---

## 1. 設計の考え方：3 層

| 層 | 役割 | 例 |
|----|------|-----|
| **第 1 層：デザイントークン** | 数値の単一ソース。色・角丸・スペーシング・コンテナを変数化 | `--section-gap`, `--container-padding-x` |
| **第 2 層：レイアウト用クラス** | 画面に依存しない汎用レイアウト。ページ・モーダル共通 | `.content-container`, `.section-stack`, `.block-stack` |
| **第 3 層：コンポーネント固有** | 見た目だけ。余白・レイアウトは第 1・2 層に任せる | `.seminar-detail-card`, `.seminar-detail-hero` |

- **レイアウト・余白** → 第 1・2 層（変数とレイアウトクラス）
- **見た目（色・角丸・シャドウ）** → 第 3 層（コンポーネント用クラス）で OK

---

## 2. ブレークポイント

Tailwind および `global.css` のメディアクエリは次の数値に揃えています。

| 名前 | 幅 | 用途 |
|------|-----|------|
| sm | 640px | 小さいタブレット以上 |
| md | 768px | タブレット以上 |
| lg | 1024px | デスクトップ（コンテナ・セクション間隔の切り替えはここ） |
| xl | 1280px | 大画面 |

`global.css` のコメントにも「Breakpoints: sm 640, md 768, lg 1024」と記載しています。変更する場合は 1 箇所で揃えてください。

---

## 3. 第 1 層：デザイントークン（変数）

### スペーシング尺度（Tailwind と揃える: 1 = 0.25rem）

| 変数 | 値 | 用途例 |
|------|-----|--------|
| --space-1 ～ --space-12 | 0.25rem ～ 3rem | アイコンと文字の間など、小さい余白 |
| --container-padding-x / -y | レスポンシブで変化 | ページの左右・上下の padding |
| --section-gap | 2rem（lg で 2.5rem） | タイトル⇔本文など、セクション間 |
| --block-gap | 1.5rem（lg で 2rem） | カード同士・ブロック同士の間隔 |
| --hero-to-detail-gap | 3rem（lg で block-gap） | ヒーローセクション ⇔ 詳細セクションの間隔（小画面＝カード間の約2倍、PC＝タイトル上と同じ高さ） |
| --hero-title-padding-inline | 2.5rem（lg で 3rem） | ヒーロー内タイトル行の左右パディング |
| --hero-title-padding-block-top / -bottom | block-gap / space-4 | ヒーロー内タイトル行の上下（2行目下の余白を小さく） |

- コンポーネントでは `pt-8` を乱発せず、**変数参照**（`padding-top: var(--section-gap)`）または **レイアウトクラス** を優先する。
- Tailwind の `gap-section` / `pt-section` は `@theme` で `--section-gap` と連携しているため、必要なときだけ使ってよい。

## 4. 第 2 層：レイアウト用クラス

| クラス名 | 役割 | 使用例 |
|----------|------|--------|
| `.content-container` | 全画面共通の max-width・横 padding・縦 padding | ページのメインラッパー、モーダル本文 |
| `.section-stack` | 縦に並ぶ「セクション」同士の間隔（gap） | タイトルブロックと本文を包む要素 |
| `.block-stack` | カードやブロック同士の間隔（gap） | カードの並び、モーダル右サイドバー内 |
| `.block-stack-tight` | カード内の小さなブロック間（gap: --space-4） | カード内の見出し・本文・リスト |
| `.hero-and-detail` | ヒーローセクション ＋ スペーサー ＋ 詳細セクションの縦並び（gap なし、スペーサーで間隔＝重なり防止） | モーダル・詳細ページの「ヒーロー ⇔ 詳細」レイアウト |
| `.hero-to-detail-spacer` | ヒーローと詳細の間のスペーサー（高さ＝--hero-to-detail-gap、flex-shrink-0） | ヒーローと詳細の重なり防止 |
| `.hero-title-row` | ヒーロー内タイトル行のパディング（左右大きく、2行目下の余白小さく） | ヒーローセクションのタイトル＋バッジを包む要素 |

- **セミナー詳細モーダルの構造**: **ヒーローセクション** → **`.hero-to-detail-spacer`** → **詳細セクション**（その中に複数カード）。ヒーロー⇔詳細の間隔は `--hero-to-detail-gap`（スペーサーで確保＝重なり防止）、詳細セクション内のカード間は `.section-stack` / `.block-stack`（`--block-gap`）で調整する。
- セミナー詳細**ページ**でも**モーダル**でも、同じ `.section-stack` / `.block-stack` を使う。
- レスポンシブは**変数側**（lg で `--section-gap` などが大きくなる）に寄せているため、クラス名は増やさない。

---

## 5. Tailwind との役割分担

| 用途 | 使うもの |
|------|----------|
| レイアウト（コンテナ・セクション間・ブロック間） | **global.css のクラス + 変数**（`.content-container`, `.section-stack`, `.block-stack`, `var(--section-gap)` など） |
| コンポーネント内の小さな余白（アイコンと文字の間など） | **Tailwind**（`gap-2`, `space-x-2` など） |

「どこを変えれば全体が変わるか」を分かりやすくするため、レイアウト系は global.css、細かい余白は Tailwind と切り分けています。

---

## 6. 運用ルール

1. **新しいスペーシングを足すとき**  
   まず「変数で表現できるか」「既存の `.section-stack` / `.block-stack` で足りるか」を確認する。

2. **「〇〇ページだけ」「モーダルだけ」のレイアウトクラスを増やさない**  
   見た目（色・角丸・シャドウ）はコンポーネント用クラスでよい。レイアウト・余白は汎用クラス + 変数に寄せる。

3. **例外で個別の `pt-8` などを使うとき**  
   「なぜレイアウトクラス／変数では足りないか」をコメントまたは PR で 1 行書く。  
   （例: 「このブロックだけデザイン指定で 3rem 必要」）

4. **新しい画面を作るとき**  
   ページの土台は `.content-container`、セクションの並びは `.section-stack`、カードやブロックの並びは `.block-stack` から検討する。
